# 🔧 ПОКРОКОВА ДІАГНОСТИКА ПРОБЛЕМИ З КНОПКАМИ

## ✅ ЩО ВЖЕ ЗРОБЛЕНО:
1. Виправлено синтаксичну помилку в `script.js` (зайва дужка)
2. Додано атрибут `defer` до всіх скриптів
3. Оновлено версію кешу до v35
4. Додано детальне логування

## 🧪 КРОК 1: ТЕСТ ПРОСТОГО HTML
1. Відкрий файл `test_buttons.html` в браузері
2. Натисни на всі 3 кнопки
3. Перевір чи з'являються повідомлення

**Якщо кнопки НЕ працюють:**
- Проблема в браузері або JavaScript вимкнено
- Перевір налаштування браузера

**Якщо кнопки працюють:**
- JavaScript працює, проблема в основному додатку
- Переходь до кроку 2

## 🔍 КРОК 2: ПЕРЕВІРКА КОНСОЛІ БРАУЗЕРА
1. Запусти сервер: `python app.py`
2. Відкрий http://localhost:5000
3. Натисни `F12` або `Ctrl+Shift+I` (відкрити DevTools)
4. Перейди на вкладку "Console"

**Що шукати:**
- ❌ Червоні помилки (errors)
- ⚠️ Жовті попередження (warnings)
- ✅ Зелені повідомлення про ініціалізацію

**Очікувані повідомлення:**
```
🚀 Запуск додатку...
📄 readyState: interactive або complete
🔧 Ініціалізація AI Асистента...
📋 Перевірка елементів:
  chatMessages: ✅
  userInput: ✅
  sendBtn: ✅
  loading: ✅
🔗 Прикріплення обробників подій...
  ✅ sendBtn обробник додано
  ✅ userInput обробники додано
✅ Ініціалізація завершена!
✅ Додаток ініціалізовано успішно!
```

## 🖱️ КРОК 3: ТЕСТ КНОПКИ ВІДПРАВКИ
1. Напиши щось в поле вводу
2. Натисни кнопку відправки (або Enter)
3. Дивись в консоль

**Очікувані повідомлення при кліку:**
```
🖱️ Клік по кнопці відправки [MouseEvent]
📤 sendMessage викликано
📝 Повідомлення: [твій текст]
```

**Якщо НЕ з'являються:**
- Обробник не прикріплений
- Елемент не знайдено
- JavaScript не завантажився

## 🧹 КРОК 4: ОЧИЩЕННЯ КЕШУ
**Метод 1: Жорстке оновлення**
- Windows/Linux: `Ctrl + Shift + R`
- Mac: `Cmd + Shift + R`

**Метод 2: Очищення через DevTools**
1. Відкрий DevTools (F12)
2. Клікни правою кнопкою на кнопку оновлення браузера
3. Вибери "Empty Cache and Hard Reload"

**Метод 3: Ручне очищення**
1. Відкрий DevTools (F12)
2. Вкладка "Application" або "Storage"
3. Знайди "Cache Storage" та "Local Storage"
4. Видали все
5. Оновіть сторінку

## 🔄 КРОК 5: ПЕРЕЗАПУСК СЕРВЕРА
```bash
# Зупини сервер (Ctrl+C)
# Запусти знову
python app.py
```

## 🌐 КРОК 6: ПЕРЕВІРКА МЕРЕЖІ
1. DevTools → вкладка "Network"
2. Оновіть сторінку
3. Знайди `script.js?v=35`
4. Перевір:
   - Status: має бути 200 (OK)
   - Size: має бути ~25-30 KB
   - Type: має бути `application/javascript`

**Якщо статус 304:**
- Це нормально (кеш)
- Але спробуй жорстке оновлення

**Якщо статус 404:**
- Файл не знайдено
- Перевір шлях до файлу

## 🐛 КРОК 7: ПЕРЕВІРКА ЕЛЕМЕНТІВ
В консолі браузера виконай:
```javascript
// Перевірка елементів
console.log('sendBtn:', document.getElementById('send-btn'));
console.log('userInput:', document.getElementById('user-input'));
console.log('window.ui:', window.ui);

// Спроба відправити повідомлення вручну
if (window.ui) {
    window.ui.userInput.value = 'Тест';
    window.ui.sendMessage();
}
```

## 🆘 ШВИДКЕ ВИПРАВЛЕННЯ
Якщо нічого не працює, спробуй:

1. Видали атрибут `defer`:
```html
<script src="{{ url_for('static', filename='script.js') }}?v=35"></script>
```

2. Або перемісти скрипти в кінець `<body>`:
```html
    </main>
    
    <script src="..."></script>
</body>
```

3. Або використай `window.onload`:
```javascript
window.onload = function() {
    const ui = new AIAssistantUI();
    window.ui = ui;
};
```
