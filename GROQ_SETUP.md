# 🚀 Налаштування GROQ API для AI Асистента

## Що таке GROQ?

GROQ - це швидкий AI сервіс, який надає доступ до потужних мовних моделей, включаючи Llama 3. Інтеграція з GROQ дозволяє асистенту давати більш розумні та контекстні відповіді.

## 🔧 Налаштування

### 1. Отримання API ключа

1. Перейдіть на [console.groq.com](https://console.groq.com)
2. Зареєструйтеся або увійдіть в акаунт
3. Перейдіть в розділ "API Keys"
4. Створіть новий API ключ
5. Скопіюйте ключ (він показується тільки один раз!)

### 2. Налаштування проекту

1. Відкрийте файл `.env` в корені проекту
2. Замініть `your_groq_api_key_here` на ваш справжній ключ:

```env
GROQ_API_KEY=gsk_your_actual_api_key_here
USE_GROQ_API=True
GROQ_MODEL=llama3-8b-8192
MAX_TOKENS=1000
```

### 3. Встановлення залежностей

```bash
pip install -r requirements.txt
```

### 4. Перезапуск сервера

```bash
python run.py
```

## 🤖 Доступні моделі

- `llama3-8b-8192` - Швидка модель (рекомендована)
- `llama3-70b-8192` - Більш потужна модель
- `mixtral-8x7b-32768` - Альтернативна модель

## 📊 Статус GROQ

В інтерфейсі асистента ви побачите індикатор статусу GROQ:

- 🤖 - GROQ активний і працює
- ⚠️ - GROQ увімкнений, але недоступний
- 🔒 - GROQ вимкнений (локальні відповіді)
- ❌ - Помилка з'єднання

## 🌟 Переваги GROQ інтеграції

### Розумніші відповіді
- Контекстне розуміння розмови
- Персоналізація на основі пам'яті користувача
- Адаптація до настрою асистента

### Аналіз настрою
- Визначення емоційного стану користувача
- Адаптація стилю спілкування

### Креативність
- Генерація оригінальних відповідей
- Космічна тематика в відповідях

## 🔄 Резервний режим

Якщо GROQ недоступний, асистент автоматично переключається на локальні відповіді:
- Використовує вбудовану базу знань
- Зберігає всі функції (пам'ять, настрої, команди)
- Продовжує працювати без перерв

## 🛠️ Налаштування моделі

Ви можете змінити модель в файлі `.env`:

```env
# Для швидших відповідей
GROQ_MODEL=llama3-8b-8192

# Для більш якісних відповідей
GROQ_MODEL=llama3-70b-8192

# Для довших контекстів
GROQ_MODEL=mixtral-8x7b-32768
```

## 📈 Моніторинг використання

1. Перейдіть на [console.groq.com](https://console.groq.com)
2. Відкрийте розділ "Usage"
3. Відстежуйте кількість запитів та токенів

## 🚨 Обмеження

- **Безкоштовний план**: обмежена кількість запитів на день
- **Швидкість**: залежить від завантаженості сервісу
- **Контекст**: максимальна довжина повідомлення

## 🔧 Налагодження

### Перевірка статусу
```bash
curl http://localhost:5000/groq-status
```

### Логи помилок
Перевірте консоль сервера на наявність помилок GROQ API.

### Тестування без GROQ
Встановіть в `.env`:
```env
USE_GROQ_API=False
```

## 💡 Поради

1. **Економте токени**: короткі запити = менше витрат
2. **Використовуйте кеш**: повторні запити можуть кешуватися
3. **Моніторте ліміти**: слідкуйте за використанням API
4. **Резервний план**: завжди тестуйте без GROQ

## 🎯 Приклади використання

```
Користувач: Розкажи про Python

```


```
Користувач: Придумай космічну історію
GROQ відповідь: Оригінальна космічна історія з персонажами та сюжетом
```
